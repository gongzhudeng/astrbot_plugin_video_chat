# 灵犀 · 视频理解

AstrBot 视频理解插件。支持直发视频，以及抖音与 B 站链接；统一提取作品信息、画面、字幕或语音原文、高赞评论和回复，再交给当前对话模型自然回答。

## 支持范围

| 输入 | 支持内容 |
|------|----------|
| 直发视频 | 自动抽帧、可选 STT；只发视频时默认概括，附带文字时按用户问题回答 |
| 抖音链接 | 视频、图文、动图帖；作品信息；高赞评论与回复；评论图片可选理解 |
| B 站链接 | BV、av、`b23.tv` 和完整链接；字幕优先；高赞评论与回复；评论图片可选理解 |

当前不读取弹幕，不支持小红书。

## 核心行为

- 默认自动解析当前消息中的直发视频，以及明确出现的抖音/B站链接。
- QQ/NapCat 直发视频会在 AstrBot 附件预处理前通过标准视频 URL 本地化，避免裸文件名导致 `not a valid file`；不依赖 NapCat 私有深层字段。
- 与消息合并器配合时，`文字 → 视频 → 文字` 会合成为包含 `[视频]` 占位和真实视频组件的一条请求，视频只解析一次。
- 直发视频、自动链接、`/视频 <链接>` 和工具 `analyze_video` 共用同一套视觉、抽帧、时长和 STT 配置。
- 普通聊天中的解析结果会注入当轮用户消息，由当前对话模型根据用户问题自然回答。
- 引用视频不会重新下载、抽帧或调用 STT/视觉模型，而是使用同一会话中此前保存的解析上下文；历史不足时需要重新发送视频。
- “最大保留完整视频上下文数量”默认是 3。直发视频和自动链接统一计数，超出后按时间顺序只替换最旧的视频详情块；用户原话、链接、同一文本块中的其他内容和 AI 回复保持不变。设置为 0 表示不限制。
- 连续发送新视频时会先为本轮解析预留一个名额。例如上限为 3，则新视频进入前最多保留 2 个历史完整详情，新解析完成后总数仍为 3。
- 视频视觉模型可参考同轮合并发送的用户文字。该文字只作为问题语境，不会被当作视频画面事实，并受最大字符数限制。
- B 站有字幕时优先使用字幕，并默认跳过视频下载、画面分析和 STT；没有字幕时才分析画面并按配置执行 STT。
- 抖音视频优先通过 `video_url` 交给视觉模型。此路径可能让模型读取完整视频和完整音轨，不受“只分析前 N 秒”限制。
- “只分析前 N 秒”仅约束本地抽帧，以及开启“语音识别跟随画面时长”后的 STT 音频裁剪。
- STT 仅提供可核验的逐字转写，最终概括由对话模型完成。关闭 STT 后不输出独立的 `【语音原文】`。
- 视觉模型和 STT 均支持“首选模型 → 单个兜底模型”，相同实例会自动去重；全部失败时省略对应区块。
- 评论按点赞数降序保留。总预算包含一级评论、回复和评论图片描述；预算不足时先移除当前评论的低优先级回复，再停止加入后续低赞评论。
- 评论图片分析默认关闭。开启后，一级评论与回复中的图片会合并为一次视觉请求；GIF、WebP 动图和视频化媒体只取一帧。失败仅省略图片描述。

## 安装

在 AstrBot 插件市场搜索 `astrbot_plugin_video_chat`，或手动安装：

```bash
cd data/plugins
git clone https://github.com/gongzhudeng/astrbot_plugin_video_chat.git
```

重启 AstrBot 后在插件配置页完成模型和平台配置。

### 依赖

插件依赖由 AstrBot 安装。以下功能还需要系统可用的 `ffmpeg`：

- 视频本地抽帧
- STT 音频提取与裁剪
- 评论区短视频媒体抽取首帧

Windows 可填写 `ffmpeg.exe` 绝对路径，Linux/macOS 可通过包管理器安装并加入 PATH。

## 配置说明

### 自动解析与上下文

| 配置项 | 说明 |
|--------|------|
| 自动解析消息中的视频 | 开启后处理顶层直发视频和明确的抖音/B站链接；引用视频不触发解析 |
| 最大保留完整视频上下文数量 | 默认 3；直发视频和自动链接统一计数，旧详情替换为标记；0 表示不限制 |
| 直发视频缓存保留时间 | 默认 3600 秒；成功解析后立即删除，异常遗留文件超时清理 |
| 直发视频缓存容量上限 | 默认 2048 MB；超限时清理最旧的已完成缓存文件 |

关闭自动解析后，仍可使用 `/视频` 命令和 `analyze_video` 工具。上下文限额仍会继续清理插件以前写入的旧视频详情。

### 视觉理解

| 配置项 | 说明 |
|--------|------|
| 视频转述模型供应商 | 首选视觉模型；留空使用当前会话模型 |
| 视频转述兜底模型 | 首选模型失败后尝试一次 |
| 转述提示词 | 用于视频画面、视频 URL 和作品图文；可自动追加同轮用户文字作为语境，不用于评论图片 |
| 视频同轮文字语境最大长度 | 默认 500 字符；0 表示不向视频视觉模型追加用户文字 |
| 评论图片分析提示词 | 独立控制评论及回复图片的描述重点和篇幅；编号映射协议由插件自动追加 |
| 允许临时下载视频 | 通用解析器是否允许下载后抽帧 |
| 抽帧速率 / 最大帧数 / 只分析前 N 秒 | 仅控制本地抽帧范围 |

评论图片提示词与视频转述提示词相互独立，可在插件设置中自由修改。

### 字幕与 STT

| 配置项 | 说明 |
|--------|------|
| B站 SESSDATA Cookie | 用于优先读取 B 站字幕 |
| 字幕同时抽帧 | 默认关闭；开启后即使有字幕也补充画面分析 |
| 启用视频语音识别 | 无字幕时提取音频并逐字转写 |
| 首选 STT 模型 / STT 兜底模型 | STT Provider 顺序 |
| 语音识别跟随画面时长 | 开启时按“只分析前 N 秒”裁剪；关闭时识别完整音频 |

### 高赞评论

| 配置项 | 说明 |
|--------|------|
| 读取高赞评论 | 控制 B 站和抖音评论读取 |
| 高赞评论最多条数 | 一级评论候选数量 |
| 每条高赞评论最多回复数 | 每条一级评论保留的回复上限 |
| 高赞评论总字数上限 | 评论正文、回复和图片描述的共同预算 |
| 分析评论区图片 | 默认关闭；关闭后不下载评论媒体、不调用视觉模型 |
| 评论图片分析提示词 | 可编辑的独立规则，不使用视频转述提示词 |
| 评论图片单次最多分析数量 | 一级评论与回复共用的批量上限 |

评论输出示例：

```text
1. [12536赞] 评论正文
   图片：图片内容概括
   - 回复（用户名）：回复正文
     图片：回复图片内容概括
```

### 抖音评论

抖音评论受实时风控影响，纯 HTTP 轻量签名可能返回 HTTP 200 空响应。需要更高成功率时，请开启“抖音评论浏览器模式”：插件会优先捕获已登录抖音页面自身发出的真实评论请求，失败后再尝试本地轻量签名。全程不调用第三方在线签名服务。

仅打开插件开关不会自动启动浏览器。Windows 下需要先关闭或另开一个 Chrome/Edge 实例，并使用独立调试用户目录启动：

```powershell
# Chrome
& "C:\Program Files\Google\Chrome\Application\chrome.exe" --remote-debugging-port=9222 --user-data-dir="D:\douyin-cdp-profile"

# Edge
& "C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe" --remote-debugging-port=9222 --user-data-dir="D:\douyin-cdp-profile"
```

然后执行：

1. 在新打开的调试浏览器窗口中登录 `douyin.com`，确认网页能正常显示评论。
2. 保持该浏览器运行。
3. 插件中开启“抖音评论浏览器模式”，CDP 地址填写 `http://127.0.0.1:9222`。
4. 浏览器访问 `http://127.0.0.1:9222/json/version` 能看到 JSON，才表示调试端口已就绪。

插件只连接现有浏览器实例，不安装或启动浏览器。浏览器关闭、未登录、页面本身看不到评论或账号被风控时，插件同样无法获取评论；评论失败不会影响作品主体解析。

## 使用方法

直接发送视频，或发送明确的视频链接：

```text
[视频附件]
这个视频里的人为什么突然停下来了？

https://v.douyin.com/xxxx/
https://www.bilibili.com/video/BV1xxxx
```

也可以使用命令：

```text
/视频 https://v.douyin.com/xxxx/
/视频 https://www.bilibili.com/video/BV1xxxx
```

支持 Function Calling 的主模型也可调用 `analyze_video`。同一条消息里的明确链接已经自动解析后，工具不会重复执行。

引用此前的视频提问时，插件不会再次解析视频。消息合并器会将引用关系标记为 `[引用视频消息]`，当前模型据此读取会话历史中的解析结果。若该详情已超过保留上限、会话已重置，或近期有多个视频导致语义不明确，请重新发送视频。

## 常见问题

**抖音作品或评论无法读取？**

作品解析可更新登录状态下导出的 Douyin Cookie。评论建议开启浏览器模式，并确认 `http://127.0.0.1:9222/json/version` 可访问、调试浏览器已登录且网页本身能显示评论。轻量签名受平台风控影响，不保证稳定可用。

**B 站没有字幕？**

并非所有视频都有字幕。确认 `SESSDATA` 有效；无字幕时插件会降级到画面分析和可选 STT。

**关闭 STT 后模型为什么仍能听到视频声音？**

支持原生视频输入的多模态模型可能自行读取完整音轨。关闭 STT 只是不再单独转写并输出 `【语音原文】`。

**提示找不到 ffmpeg？**

安装 ffmpeg 并加入 PATH，或在插件配置中填写可执行文件绝对路径。
